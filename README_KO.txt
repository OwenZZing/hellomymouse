# 🔭 Hypothesis Maker — 사용 설명서 (한국어)

> 연구실 논문 PDF → AI 분석 → Research Starter Kit (.docx) 자동 생성  
> Made by @hellomymouse · kby930@gmail.com

---

## 이게 뭔가요?

연구실에 처음 들어간 대학원생(또는 학부 인턴)을 위한 도구입니다.  
연구실 논문 PDF를 폴더에 넣으면, AI가 자동으로 분석하여 아래 내용이 담긴 Word 문서를 만들어줍니다:

- 연구실 프로젝트 목록 및 개요
- 연구란 무엇인가? (학부생 눈높이 입문 설명)
- 연구실 보유 기술 / 장비 목록
- 논문별 요약 (핵심 결과 + 한계점)
- 가설 후보 H1~H7 (안전한 가설 6개 + 최신 트렌드 가설 1개)
  - 각 가설의 별점 임팩트 / 예상 기간 / 평가 지표 / Baseline / Fallback Plan
- 소요 비용 추정 (KRW 기준)
- PI 확인 체크리스트
- Background Knowledge Guide (핵심 개념 / 키워드 / 추천 저널)
- 첫 3개월 Roadmap

---

## ⚠ 분석 전 반드시 선배에게 물어보세요

AI는 논문에 적힌 내용만 보고 분석합니다. **논문과 현실 사이에는 큰 간극이 있습니다.**  
가설을 실행하기 전에 반드시 선배 또는 교수님께 아래를 확인하세요:

| 확인 항목 | 이유 |
|-----------|------|
| **논문에 나온 로봇 / 장비 / 센서가 현재도 랩에 있나요?** | 시간이 지나면서 업그레이드·교체·노후화될 수 있습니다 |
| **그 장비를 내가 사용할 수 있나요?** | 다른 프로젝트에 쓰이거나 다른 로봇 제작에 부품이 이전됐을 수 있습니다 |
| **실험 환경(서버, GPU, 시뮬레이터 라이선스)은 갖춰져 있나요?** | 인프라가 바뀌었을 수 있습니다 |
| **이 가설이 현재 랩의 방향과 맞나요?** | AI가 만든 가설은 어디까지나 후보입니다 |

> 장비 사용이 어렵다면, 보고서의 **Fallback Plan** 항목에서 시뮬레이터(Isaac Gym, Gazebo, MuJoCo 등) 대체 방안을 참고하세요.

## API 크레딧이 없을 때 — Claude Code로 수동 생성

API 한도 초과 또는 크레딧 부족으로 프로그램을 돌릴 수 없을 때,
Claude Code에 논문 초록을 직접 붙여넣으면 동일한 포맷의 리포트를 수동으로 만들 수 있습니다.

방법:
1. 논문 초록(Abstract)들을 복사합니다
2. Claude Code에 아래처럼 요청합니다:

    배정 프로젝트: [프로젝트명]
    아래 논문 초록을 바탕으로 Research Starter Kit 리포트를 만들어줘.
    Papers 폴더: [폴더 경로]
    [초록 붙여넣기]

3. Claude가 리포트 생성용 일회용 스크립트를 작성해줍니다 (레포 루트가 아닌 별도 폴더, 예: scratch/ 에 저장)
4. 스크립트를 실행하면 .docx 파일이 자동 생성됩니다:

    python scratch/your_report_script.py

참고: 과거 루트에 있던 generate_report_*.py 는 ../legacy_archive/ 로 이동되었습니다. 현재 앱과 drift가 있어 실행하지 마세요.

비교:
  항목                    프로그램 (API)       Claude Code 수동
  논문 전체 분석          PDF 전체 파싱        초록 위주
  속도                    3~10분 자동          Claude 응답 후 즉시
  비용                    API 크레딧 소모      없음 (Claude Code 구독)
  출력 포맷               동일 (.docx)         동일 (.docx)

## 설치 방법

### 방법 A: exe 파일 (권장, 비개발자)

`HypothesisMaker.exe` 파일을 더블클릭하면 바로 실행됩니다.  
별도 설치 필요 없습니다.

### 방법 B: Python으로 직접 실행 (개발자)

```bash
# Python 3.11 이상 필요
cd Hypothesis-Finder
python -m venv venv
source venv/Scripts/activate        # Windows
pip install -r requirements.txt
python main.py
```

## 사용 방법

### Step 1 — 논문 준비
연구실의 최근 5년 논문 PDF를 최대한 모아 하나의 폴더에 넣습니다.  
**10~20편** 정도가 적당합니다. 너무 적으면 분석 품질이 낮아집니다.

### Step 2 — API Key 입력
프로그램 상단에서 AI를 선택하고 API Key를 입력합니다.

| AI | API Key 발급 | 비용 (논문 10편 기준) |
|----|------------|----------------------|
| **Claude (고품질)** | [console.anthropic.com](https://console.anthropic.com) → API Keys | ~$0.3~0.8 (₩400~1,100) |
| **OpenAI GPT-4o** | [platform.openai.com](https://platform.openai.com) → API keys | ~$0.5~1.5 (₩700~2,000) |
| **Gemini Flash (저렴/무료)** | [aistudio.google.com](https://aistudio.google.com) → Get API key | ~$0.05~0.2 (₩70~280) |

> 💡 **비용이 걱정된다면 Gemini Flash를 추천합니다.**  
> Google AI Studio에서 무료로 API Key를 발급받을 수 있습니다.

> API Key는 로컬에 저장됩니다 (`~/.hypothesis_maker_config.json`).  
> 공용 컴퓨터 사용 후에는 Key를 삭제하세요.

### Step 3 — 논문 폴더 선택
"연구실 논문 폴더" 옆 **찾기** 버튼으로 폴더를 선택합니다.

### Step 4 — (선택) 추가 참조 논문
교수님이 "이 논문도 봐"라고 추천하신 논문이 있다면, 별도 폴더에 넣고 "추가 참조 논문 폴더"에서 선택합니다.

### Step 5 — 프로젝트 목록 파악 (Stage 0)
**"🔍 프로젝트 목록 파악"** 버튼을 클릭합니다.  
AI가 논문 제목·초록만 빠르게 읽어 프로젝트 목록을 추출합니다. (30초~1분)

### Step 6 — 배정된 프로젝트 선택
- 교수님이 특정 프로젝트를 배정하셨다면 **"목록에서 선택"**으로 선택하거나 직접 입력하세요.
- 배정이 없다면 비워두면 모든 프로젝트를 고르게 분석합니다.

### Step 7 — (선택) 교수님 지시사항 입력
교수님이 말씀하신 방향, 선호 방법론, 참고 논문 등을 자유롭게 입력합니다.

### Step 8 — 분석 시작
**"▶ 분석 시작"** 버튼을 클릭합니다.  
논문 수에 따라 3~10분 소요됩니다. 진행 상황이 아래 로그에 표시됩니다.

### Step 9 — 저장 및 열기
분석이 완료되면 저장 위치를 선택하고 **"📄 리포트 생성"**을 클릭합니다.

## 가설 임팩트 별점 가이드

| 별점 | 의미 |
|-----|-----|
| ★☆☆☆☆ | 졸업 아슬아슬. 심사위원에 많이 깎임 |
| ★★☆☆☆ | 졸업 무난. 취직은 다른 스펙이 받쳐줘야 함 |
| ★★★☆☆ | 졸업 안정적. 취직 시 논문으로 설명 가능 |
| ★★★★☆ | 졸업 오래 걸릴 수 있음. 끝나면 포지션 골라감 |
| ★★★★★ | 분야를 바꾸는 논문. 5년이 넘어도 worth it |

> ※ 예상 기간은 해당 기술을 이미 숙지한 경우 기준입니다.  
> 신기술 학습 곡선을 포함하면 실제로는 **2~3배** 소요될 수 있습니다.
